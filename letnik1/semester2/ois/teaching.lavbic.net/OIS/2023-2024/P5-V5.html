Poglavje 11 P5 → V5 Povezovanje na podatkovno bazo

P5 → V5

V pripravah na vaje V5 bomo razširili arhitekturo odjemalec/strežnik s podatkovno bazo, kjer bomo spoznali naslednje koncepte:

  • Chinook zbirko podatkov,
  • master-detail načrtovalski vzorec,
  • podatkovna baza SQLite,
  • Node.js knjižnica Express.js.

11.1 Priprava okolja

Celotna izvorna koda primera v nadaljevanju je na voljo v GitHub repozitoriju P5-V5. Najprej na GitHub izdelajte kopijo projekta (angl. Fork) z istim imenom repozitorija, nato pa si lokalno okolje nastavite z naslednjima ukazoma:

git clone git@github.com:{VAŠE-GITHUB-UPORABNIŠKO-IME}/P5-V5.git
cd P5-V5

ali

git clone https://github.com/{VAŠE-GITHUB-UPORABNIŠKO-IME}/P5-V5.git
cd P5-V5

11.2 Razširitev arhitekture odjemalec/strežnik s podatkovno bazo

  • predstavitvena raven (spletna stran na strani odjemalca),
  • aplikacijski strežnik (poslovna logika, ki se izvaja na strežniku),
  • podatkovna baza (persistenčna raven, kamor se shranijo podatki).

11.3 Chinook podatkovna baza

Uporabljali bomo testno Chinook podatkovno bazo (glej shemo na sliki 11.1), ki so jo v izobraževalne namene pripravili pri podjetju Microsoft.

Slika 11.1: Podatkovna shema zbirke podatkov Chinook PlantUML.

Podatkovna baza podpira digitalno spletno trgovino in vključuje transakcije, ki podpirajo spletno prodajo pesmi (npr. v iTunes). Je alternativa popularni izobraževalni podatkovni bazi Northwind.

11.4 Poizvedovalni jezik SQL

Poizvedovalni jezik SQL je de-facto standard za poizvedovanje po relacijskih podatkovnih bazah. Več o njem si bomo pogledali pri načrtovanju informacijskih sistemov. Zaenkrat bomo uporabljali zgolj poizvedbe, ki so v poenostavljeni obliki sestavljene iz naslednjih gradnikov:

SELECT atribut1, atribut2, ...
FROM tabela1, tabela2, ...
WHERE pogoj1 AND pogoj2 OR pogoj3 AND ...

Opomba: Podrobneje se boste s poizvedovalnim jezikom SQL srečali v višjih letnikih, predvsem pri predmetu Osnove podatkovnih baz.

11.5 Podatkovna baza SQLite

Gre za zelo enostavno implementacijo odprtokodne podatkovne baze SQLite, ki ob začetku uporabe ne potrebuje nobenih nastavitev, podatke pa lahko enostavno shranjuje v datoteko.

11.5.1 Vsebina podatkovne baze

Za pregled vsebine podatkovne baze lahko uporabimo orodje DB Browser for SQLite, ki je na voljo za vsa okolja (Windows, OSX in Linux). Za Windows okolje je na voljo tudi prenosna različica (tj. PortableApp), ki vam je ni potrebno namestiti in jo samo prenesete k sebi na računalnik ter poženete.

Slika 11.2: Uporabniški vmesnik DB Browser for SQLite

11.5.2 Namestitev knjižnic

Spremembe v repozitoriju od tega koraka naprej so na voljo na uveljavitvi s sporočilom Dodajanje odvisnosti od knjižnic in podatkovne baze Chinook.

Najprej moramo v okolju Node.js poskrbeti za namestitev knjižnice sqlite3, zato pripravimo package.json z naslednjo vsebino:

package.json

{
  "name": "pb",
  "description": "Povezovanje na podatkovno bazo",
  "author": "Dejan Lavbič <Dejan@lavbic.net>",
  "dependencies": {
    "sqlite3": "*"
  },
  "private": true
}

in poženemo ukaz

$ npm install

V projekt vključimo tudi datoteko Chinook.sl3, ki predstavlja zbirko podatkov obliki v SQL Lite podatkovne baze.

Namig: Če imate pri uporabi MacOS težave z namestitvijo knjižnice, si lahko pomagate z dodatnimi navodili.

Dodajmo tudi datoteko .gitignore, kjer zahtevamo, da se vsebina mape node_modules ne posreduje v oddaljeni GitHub repozitorij, saj tam ne želimo shranjevati knjižnic, ampak zgolj izvorno kodo.

.gitignore

node_modules

);

11.8.2 Funkcija, ki v povratnem klicu vrača seznam žanrov

Z namenom implementacije učinkovitega dostopa do podatkovne baze, ki omogoča več hkratnih dostopov, bomo implementirali funkcijo vrniVseZanre(povratniKlic), ki preko funkcije povratniKlic vrne seznam žanrov.

streznik_pesmi_po_zanrih.js

/**
 * Vrni seznam žanrov v povratnem klicu
 *
 * @param povratniKlic rezultat klica funkcije, ki predstavlja tabelo vrstic
 * in v vsaki vrstici je podatek o nazivu žanra
 */
const vrniVseZanre = (povratniKlic) => {
  pb.all("SELECT * FROM Genre", (napaka, vrstice) => {
    let rezultat = "Prišlo je do napake!";
    if (!napaka) {
      rezultat = "<h1>" + "Žanri" + "</h1>" + "<ul>";
      vrstice.forEach((vrstica) => {
        rezultat += "<li>" + vrstica.Name + "</li>";
      });
      rezultat += "</ul>";
    }
    povratniKlic(rezultat);
  });
};

11.8.5 Dodajanje seznama pesmi (detail)

11.8.5.1 Ob kliku se kliče nova spletna stran

Najprej dodamo povezave na vsak žanr, ki pošlje zahtevo na novo stran http://.../kategorija/{id žanra}.

streznik_pesmi_po_zanrih.js

const vrniVseZanre = (povratniKlic) => {
  pb.all("SELECT * FROM Genre", (napaka, vrstice) => {
    let rezultat = "Prišlo je do napake!";
    if (!napaka) {
      rezultat = "<h1>" + "Žanri" + "</h1>" + "<ul>";
      vrstice.forEach((vrstica) => {
        rezultat +=
          "<li>" +
          "<a href='/kategorija/" +
          vrstica.GenreId +
          "'>" +
          vrstica.Name +
          "</a>" +
          "</li>";
      });
      rezultat += "</ul>";
    }
    povratniKlic(rezultat);
  });
};

11.8.5.2 Funkcija, ki vrne seznam pesmi izbranega žanra

Seznam vseh izbranih pesmi nam vrne SQL poizvedba: SELECT * FROM Track WHERE GenreId = $id, kjer je $id identifikator žanra, ki se posreduje ob kliku na žanr v prejšnjem koraku.

streznik_pesmi_po_zanrih.js

/**
 * Vrni vse pesmi podanega žanra v povratnem klicu
 *
 * @param idZanra ID žanra
 * @param povratniKlic rezultat klica funkcije, ki predstavlja HTML kodo
 * v obliki alinej, kjer je v vsaki podatek o nazivu in ceni pesmi
 */
const vrniPesmiZanra = (idZanra, povratniKlic) => {
  pb.all(
    "SELECT * FROM Track WHERE GenreId = $id",
    { $id: idZanra },
    (napaka, vrstice) => {
      let rezultat = "Prišlo je do napake!";
      if (!napaka) {
        rezultat = "<h2>" + "Pesmi" + "</h2>" + "<ul>";
        vrstice.forEach((vrstica) => {
          rezultat +=
            "<li>" +
            "<b>" +
            vrstica.Name +
            "</b>" +
            " @ " +
            "$" +
            vrstica.UnitPrice +
            "</li>";
        });
        rezultat += "</ul>";
      }
      povratniKlic(rezultat);
    }
  );
};

11.8.5.3 Klic funkcije ob zahtevi seznama pesmi

Ob zahtevi prikaza seznama pesmi izbranega žanra prikažemo:

  • seznam vseh žanrov in
  • seznam pesmi izbranega žanra.

Uporabimo lahko preprosto gnezdenje funkcij, kjer najprej zahtevamo seznam vseh žanrov, nato v povratnem klicu še seznam pesmi izbranega žanra in na koncu vse skupaj prikažemo na strani.

streznik_pesmi_po_zanrih.js

streznik.get("/kategorija/:idKategorije", (zahteva, odgovor) => {
  vrniVseZanre((rezultatMaster) => {
    vrniPesmiZanra(zahteva.params.idKategorije, (rezultatDetail) => {
      odgovor.send(rezultatMaster + rezultatDetail);
    });
  });
});

11.8.5.4 Nadgradnja seznama pesmi z izvajalcem

Spremenimo izpis seznama pesmi tako, da poleg naslova pesmi in cene dodamo še izvajalca.

streznik_pesmi_po_zanrih.js

/**
 * Vrni vse pesmi, skupaj z avtorji, podanega žanra v povratnem klicu
 *
 * @param idZanra ID žanra
 * @param povratniKlic rezultat klica funkcije, ki predstavlja HTML kodo
 * v obliki alinej, kjer je v vsaki podatek o nazivu, izvajalcu in ceni
 */
const vrniPesmiInAvtorjeZanra = (idZanra, povratniKlic) => {
  pb.all(
    "SELECT Track.Name AS pesem, Artist.Name AS izvajalec, \
            Track.UnitPrice AS cena \
    FROM    Track, Album, Artist \
    WHERE   Track.AlbumId = Album.AlbumId AND \
            Album.ArtistId = Artist.ArtistId AND \
            GenreId = $id",
    { $id: idZanra },
    (napaka, vrstice) => {
      let rezultat = "Prišlo je do napake!";
      if (!napaka) {
        rezultat = "<h2>" + "Pesmi" + "</h2>" + "<ul>";
        vrstice.forEach((vrstica) => {
          rezultat +=
            "<li>" +
            "<b>" +
            vrstica.pesem +
            "</b>" +
            " (" +
            vrstica.izvajalec +
            ")" +
            " @ " +
            "$" +
            vrstica.cena +
            "</li>";
        });
        rezultat += "</ul>";
      }
      povratniKlic(rezultat);
    }
  );
};

Spremenimo tudi funkcijo, ki se kliče ob zahtevi podrobnosti žanra.

streznik_pesmi_po_zanrih.js

streznik.get("/kategorija/:idKategorije", (zahteva, odgovor) => {
  vrniVseZanre((rezultatMaster) => {
    vrniPesmiInAvtorjeZanra(zahteva.params.idKategorije, (rezultatDetail) => {
      odgovor.send(rezultatMaster + rezultatDetail);
    });
  });
});
